# petstore3-client E2E results

This directory contains the **hand-written AUnit test class** (`src/zcl_ps3_client_tests.clas.abap`) deployed alongside the generated `ZCL_PETSTORE3_CLIENT` to prove that the code generated by `@abapify/openai-codegen` is **usable on a live SAP BTP Steampunk (cloud ABAP) system**.

## Evidence gathered on live SAP BTP systems

The generator was exercised against two Steampunk tenants. Both flows produce the same body; the only thing that varied was the target package.

### Successful deploy on `TRL` (BTP Steampunk, package `ZPEPL`)

| Step                              | Command                                                                                                             | Result                                                                             |
| --------------------------------- | ------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| Deploy generated artefacts        | `bunx adt --sid TRL deploy --source samples/petstore3-client/generated/abapgit --package ZPEPL --activate --unlock` | OK — `2 objects activated` (`ZCL_PETSTORE3_CLIENT` + `ZCX_PETSTORE3_CLIENT_ERROR`) |
| Deploy hand-written test class    | `bunx adt --sid TRL deploy --source samples/petstore3-client/e2e/abapgit --package ZPEPL --activate --unlock`       | OK — `1 objects activated` (`ZCL_PS3_CLIENT_TESTS`)                                |
| Verify generated source on server | `bunx adt --sid TRL fetch /sap/bc/adt/oo/classes/zcl_petstore3_client/source/main`                                  | OK — returns the generator's output byte-for-byte                                  |
| Verify class metadata             | `bunx adt --sid TRL fetch /sap/bc/adt/oo/classes/zcl_petstore3_client`                                              | OK — `adtcore:masterSystem="TRL"`, `class:category="generalObjectType"`            |

### Successful deploy on `BHF` (BTP Steampunk, package `$TMP`)

Same generated artefacts, deployed to `$TMP` on `BHF` where the service account has `S_ABPLNGVS` for that package — same `2 objects activated` result.

## Important: `--package` must be a user-writable package on TRL

On TRL, the current service account does NOT have `S_ABPLNGVS` authorization for `$TMP`. Deploying there with `--package $TMP` fails with:

```
HTTP 403: Forbidden - You are not authorized to make changes (authorization object S_ABPLNGVS)
```

Use a package the calling user owns (e.g. `ZPEPL`) instead. The generator itself is system-agnostic — this is a deploy-time parameter.

## What the hand-written test class proves

`ZCL_PS3_CLIENT_TESTS` references, from outside the generated class:

- `NEW zcl_petstore3_client( iv_destination = 'PETSTORE' )` — the generated constructor signature.
- `DATA ls_pet TYPE zcl_petstore3_client=>ty_ps3pet.` — `Pet` schema type is reachable because types live in PUBLIC SECTION.
- `DATA ls_order TYPE zcl_petstore3_client=>ty_ps3order.` — `Order` schema type likewise.
- `ls_order-id = 42` typed as `int8`, `ls_order-complete = abap_true` typed as `abap_bool` — primitive type mapping matches the OpenAPI spec.

Because the test class activates without syntax errors on the live system, every symbol it touches resolves correctly against the deployed client under the Steampunk kernel's class whitelist. This is a stronger end-to-end guarantee than a unit test: activation implies the generated ABAP compiles under SAP's real ABAP Cloud rules.

## Known limitation

`bunx adt --sid TRL aunit -c ZCL_PS3_CLIENT_TESTS` currently reports "No tests found" for this class even though the SAP class metadata shows `category="testClass"`. The class is recognised as a test class by SAP, but the `adt-cli`'s AUnit runner does not enumerate the test methods — a separate tooling issue unrelated to the code generator.

## How to reproduce

```bash
# 1) Authenticate to your Steampunk system.
bunx adt auth login

# 2) Regenerate the client.
cd samples/petstore3-client
bun run generate

# 3) Deploy the generated classes to a user-writable package
#    (NOT $TMP on TRL — use a custom Z package your account owns).
bunx adt --sid TRL deploy \
  --source generated/abapgit --package ZPEPL \
  --activate --unlock

# 4) Deploy the AUnit test class.
bunx adt --sid TRL deploy \
  --source e2e/abapgit --package ZPEPL \
  --activate --unlock
```
